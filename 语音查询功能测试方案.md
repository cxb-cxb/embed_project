# 客户语音查询与智能回复测试方案

## 测试目标

验证“客户语音查询与智能回复”第一阶段功能是否可用于演示：

- 离线关键词 + 模板回复可稳定工作。
- 终端输入问题后能返回正确商品信息。
- 可通过 `CurrentProduct` 模拟视觉识别结果。
- `auto` 模式在未配置大模型 API Key 时能自动降级。
- 后续接入麦克风 ASR 和扬声器 TTS 时，核心问答逻辑不需要重写。

进入项目目录：

```powershell
cd "C:\Users\33591\Documents\嵌赛 2"
```

确认脚本语法：

```powershell
python -m py_compile tools\voice_retail_assistant.py
```

## 测试 1：离线单句价格查询

命令：

```powershell
.\start_voice_assistant_demo.ps1 -Mode offline -Question "牛奶多少钱"
```

预期结果：

```text
助手> 牛奶售价 4.5 元。
[reply_source=offline]
```

通过标准：商品名称和价格正确。

## 测试 2：离线商品列表查询

命令：

```powershell
.\start_voice_assistant_demo.ps1 -Mode offline -Question "现在有哪些商品"
```

预期结果：

```text
助手> 当前已录入商品有：可乐、方便面、薯片、饼干、牛奶、面包、牙膏、矿泉水、纸巾、肥皂。
```

通过标准：10 类商品全部出现。

## 测试 3：当前视觉识别商品联动

模拟当前视觉识别结果为 `milk`：

```powershell
.\start_voice_assistant_demo.ps1 -Mode offline -CurrentProduct milk -Question "这个商品还有库存吗"
```

预期结果：

```text
助手> 牛奶当前库存 14 件，库存充足。
```

通过标准：问题中没有直接说“牛奶”，但系统能根据 `CurrentProduct` 回答牛奶库存。

## 测试 8：联网大模型增强模式

如果有 OpenAI 兼容 API，先设置环境变量：

```powershell
$env:RETAIL_LLM_API_KEY="你的API_KEY"
$env:RETAIL_LLM_BASE_URL="https://api.openai.com/v1"
$env:RETAIL_LLM_MODEL="gpt-4o-mini"
```

运行：

```powershell
.\start_voice_assistant_demo.ps1 -Mode auto -CurrentProduct milk -Question "这个适合早餐吗"
```

预期结果：

```text
[reply_source=llm]
```

通过标准：

- 回复内容围绕牛奶。
- 语气自然。
- 没有编造目录外商品信息。
- 输出中出现 `[reply_source=llm]`。

## 测试 10：视觉识别与语音查询联动

先启动视觉识别：

```powershell
.\start_visual_lvds_demo.ps1
```

确认终端持续输出类似：

```text
0001: milk 牛奶 confidence=0.990
```

另开一个 PowerShell，启动语音查询：

```powershell
.\start_voice_mic_query_loop.ps1 -Seconds 6 -ReplyMode auto
```

对着麦克风问：

```text
这个多少钱
```

通过标准：

- 终端先显示 `当前视觉商品>`。
- ASR 能识别出“这个多少钱”或接近文本。
- 助手回复当前摄像头识别商品的价格，而不是要求手动指定商品。

## 最终演示验收

运行：

```powershell
.\start_final_demo.ps1 -VoiceSeconds 6 -ReplyMode auto
```

通过标准：

- 自动打开视觉识别窗口和语音查询窗口。
- 视觉窗口持续识别当前商品。
- 语音窗口能识别“这个多少钱”并回复。
- LVDS 信息区出现最近一次 `VOICE QA`。
- 切换 `-ReplyMode offline` 后仍能完成保底回复。
