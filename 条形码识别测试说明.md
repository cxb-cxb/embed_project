# 条形码识别测试说明

## 当前状态

- 已在 `qr_scanner_display` 中加入 EAN-13 条形码识别。
- 已部署到开发板：`/userdata/Embed_project/bin/qr_scanner_display`。
- 已支持真实商品条形码映射表：`data/barcode_aliases.csv`。
- 条形码识别后沿用现有购物车逻辑，LVDS 顶部商品信息和购物车状态会同步更新。

## 使用真实商品条形码

真实包装条形码需要先绑定到 10 类商品之一。例如真实牛奶包装上的条码是 `692xxxxxxxxxx`，执行：

```powershell
cd "C:\Users\33591\Documents\嵌赛 2"
.\register_real_barcode.ps1 -ProductId milk -Barcode 692xxxxxxxxxx
```

可用商品 ID：

```text
water, cola, milk, bread, noodle, chips, biscuit, toothpaste, tissue, soap
```

绑定后重新启动扫码程序即可生效。

## 启动命令

在电脑 PowerShell 中执行：

```powershell
adb shell
cd /userdata/Embed_project
sh scripts/qr_realtime_lvds.sh
```

也可以继续使用总启动脚本：

```powershell
cd "C:\Users\33591\Documents\嵌赛 2"
.\start_final_demo.ps1 -VoiceSeconds 6 -ReplyMode auto
```

## 测试方法

1. 先用 `register_real_barcode.ps1` 录入真实包装条形码。
2. 启动扫码程序。
3. 将商品包装上的条形码横向放入摄像头画面中部。
4. 尽量保持条纹清晰、不要大角度倾斜。

## 预期结果

终端识别到真实条形码时会显示：

```text
>>> BARCODE: 692xxxxxxxxxx <<<
    added:Milk total:4.50
```

LVDS 屏幕会同步更新商品信息和购物车状态。

## 注意事项

- 当前摄像头条形码识别支持 EAN-13，也就是国内商品包装上最常见的 13 位标准商品条码。
- 项目商品表仍保留原来的 10 类商品，真实条码通过 `barcode_aliases.csv` 绑定到商品类别。
- 同一个条形码一直停在画面里只计数一次；拿出画面后再次放入，会再次计数。
- 如果识别不稳定，把条形码放大、提高亮度、让条纹保持水平。
- 如果扫到条码但显示 `NO PRODUCT MAP`，说明条码识别到了，但还没有绑定到商品 ID。
