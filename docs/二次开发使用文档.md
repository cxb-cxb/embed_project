# Embed_project 二次开发使用文档

## 1. 项目目标

本项目对应赛题“选题二：端侧 AI 解决方案”中的智能零售系统设计。目标是在 QSM368ZP-WF 开发板上实现一个可演示、可扩展的端侧智能零售原型：

- 通过摄像头、条码、二维码或图像识别获得商品信息。
- 至少支持 10 种常见商品的识别、价格匹配和总价核算。
- 生成支付链接，后续可替换为微信、支付宝或学校指定支付接口。
- 支持基础语音交互，例如加入商品、查询价格、查看购物车、结算。
- 为 RKNN 模型、OpenCV 图像处理、真实语音识别和真实支付接口预留代码入口。

## 2. 项目结构

```text
Embed_project/
  CMakeLists.txt                 CMake 构建脚本
  data/products.csv              商品数据库，默认 10 种商品
  include/product.hpp            商品结构体
  include/catalog.hpp            商品目录接口
  include/cart.hpp               购物车和订单快照接口
  include/recognizer.hpp         商品识别接口
  include/payment.hpp            支付链接生成接口
  include/voice.hpp              语音命令解析接口
  src/*.cpp                      核心实现
  src/main.cpp                   命令行演示程序
  tests/test_core.cpp            核心功能测试
  scripts/camera_preview_lvds.sh 摄像头 0 到 LVDS 屏幕预览脚本
  scripts/run_on_board.sh        板端运行脚本
```

## 3. 如何写代码

### 3.1 添加或修改商品

编辑 `data/products.csv`：

```csv
id,display_name,barcode,price_cents,visual_keywords
cola,Cola,690100000002,350,cola|soda|red|can
```

字段含义：

- `id`：程序内部使用的商品 ID，建议只用英文、数字和下划线。
- `display_name`：界面显示名称。
- `barcode`：条形码或自定义编号。
- `price_cents`：价格，单位是分，例如 350 表示 3.50 元。
- `visual_keywords`：外观识别关键词，多个关键词用 `|` 分隔。

### 3.2 接入真实条码或二维码识别

当前 `RetailRecognizer::recognize()` 支持三种输入：

```cpp
RecognitionInput input;
input.barcode = "690100000002";
input.qr_payload = "product:cola";
input.visual_label = "red soda can";
```

如果后续使用 zbar、OpenCV QRCodeDetector 或摄像头扫码模块，只需要把识别出的条码字符串填入 `barcode`，或者把二维码内容填入 `qr_payload`。

二维码建议格式：

```text
product:cola
```

### 3.3 接入 RKNN 商品外观识别

当前外观识别是关键词模拟：

```cpp
recognizer.recognize({"", "", "red soda can"}, catalog);
```

后续接入 RKNN 时，建议流程如下：

1. 摄像头采集一帧图像。
2. OpenCV 做 resize、颜色格式转换、归一化。
3. RKNN 推理得到类别 ID 和置信度。
4. 将类别 ID 映射到 `Product.id`。
5. 返回 `RecognitionResult{product, confidence, "rknn"}`。

建议新建文件：

```text
include/rknn_product_detector.hpp
src/rknn_product_detector.cpp
```

然后在 `src/main.cpp` 或你的 GUI 程序中调用检测器，把结果加入购物车。

### 3.4 修改语音命令

语音命令解析位于：

```text
include/voice.hpp
src/voice.cpp
```

当前支持这些意图：

- `AddProduct`：加入商品，例如 `add cola`
- `QueryPrice`：查询价格，例如 `price milk`
- `Checkout`：结算，例如 `checkout`
- `ClearCart`：清空购物车，例如 `clear`
- `ShowCart`：查看购物车，例如 `cart`

如果使用离线语音识别模块，先把语音转成文本，再调用：

```cpp
auto action = VoiceInterpreter::parse(text, catalog);
```

### 3.5 替换真实支付接口

支付逻辑位于：

```text
include/payment.hpp
src/payment.cpp
```

当前生成的是演示 URL：

```text
https://pay.example.local/qsm368?merchant=student-team&order=...&amount=...
```

正式比赛演示时，可以保留这个链接作为“支付二维码内容”。如果需要接入真实支付，需要在 `PaymentGenerator::create()` 中替换为实际后端接口返回的支付 URL。

## 4. 编译方法

### 4.1 Linux 主机或板端 CMake 编译

```bash
cd Embed_project
cmake -S . -B build
cmake --build build
ctest --test-dir build --output-on-failure
```

生成文件：

```text
build/embed_project
build/test_core
```

### 4.2 板端无 CMake 时直接用 g++

```bash
cd Embed_project
mkdir -p build
g++ -std=c++17 -Iinclude src/*.cpp -o build/embed_project
g++ -std=c++17 -Iinclude tests/test_core.cpp src/catalog.cpp src/cart.cpp src/payment.cpp src/recognizer.cpp src/voice.cpp -o build/test_core
./build/test_core
```

### 4.3 交叉编译

如果厂商 SDK 提供 aarch64-linux-gnu 工具链，可使用：

```bash
cd Embed_project
mkdir -p build
aarch64-linux-gnu-g++ -std=c++17 -Iinclude src/*.cpp -o build/embed_project
```

如果使用 CMake toolchain 文件：

```bash
cmake -S . -B build -DCMAKE_TOOLCHAIN_FILE=/path/to/toolchain.cmake
cmake --build build
```

## 5. 推送到开发板运行

电脑已能使用 adb 时：

```bash
adb devices
adb push Embed_project /userdata/Embed_project
adb shell
cd /userdata/Embed_project
chmod +x scripts/*.sh
```

如果板端已经编译好：

```bash
./build/embed_project data/products.csv
```

如果要先在板端编译：

```bash
mkdir -p build
g++ -std=c++17 -Iinclude src/*.cpp -o build/embed_project
./build/embed_project data/products.csv
```

## 6. 摄像头 0 + LVDS 屏幕预览

你当前板子已验证摄像头 0 和 LVDS 屏幕可亮。可以使用：

```bash
sh scripts/camera_preview_lvds.sh
```

脚本内容等价于：

```bash
gst-launch-1.0 -v \
  v4l2src device=/dev/video-camera0 ! \
  video/x-raw,width=640,height=480,format=NV12,framerate=30/1 ! \
  videoconvert ! \
  video/x-raw,format=BGRx ! \
  kmssink driver-name=rockchip connector-id=154 plane-id=73 force-modesetting=true fullscreen=true sync=false
```

如果画面不显示，优先检查：

- `/dev/video-camera0` 是否存在。
- LVDS 屏幕是否已亮。
- `connector-id` 和 `plane-id` 是否因为固件变化而改变。
- `gst-launch-1.0` 和 Rockchip GStreamer 插件是否存在。

2026-07-04 屏幕修复固件验证记录：

- 只刷入修复包中的 `boot.img` 和 `uboot.img`。
- 修复后系统内核显示为 `Linux rockchip 4.19.232 #1 Thu Apr 23 10:15:10 +03 2026`。
- LVDS 状态为 `connected`，模式为 `800x1280`。
- 背光节点 `backlight`、`backlight1`、`backlight2` 均可拉到 `255`。
- 当前可用显示参数为 `connector-id=154`、`plane-id=73`、`format=BGRx`。
- 旧参数 `plane-id=89` + NV12 在该修复固件下会出现 `failed to configure video mode`。

## 7. 固件烧录说明

如果开发板系统损坏或无法启动，可以重新烧录完整固件。一般流程：

1. 安装 Rockusb 驱动。
2. 打开瑞芯微开发工具 RKDevTool。
3. 让开发板进入 Loader 模式。
4. 在工具中加载固件目录里的 `parameter.txt` 和各分区镜像。
5. 常规重刷建议选择 `uboot`、`misc`、`boot`、`recovery`、`rootfs`、`oem` 等必要分区。
6. 谨慎处理 `userdata`、`persist`、`nvdata`，它们可能包含用户数据或设备校准信息。
7. 下载完成后断电重启，并把 BOOT 开关切回正常启动位置。

你之前已经完成过一次下载，如果只是更新本项目程序，通常不需要重新烧固件，直接用 adb push 即可。

## 8. 演示流程建议

比赛演示可以按以下顺序：

1. 启动 LVDS 屏幕和摄像头预览，证明硬件链路正常。
2. 运行 `embed_project`。
3. 输入 `list` 展示 10 种商品。
4. 输入 `scan 690100000002` 模拟条码识别可乐。
5. 输入 `see red soda can` 模拟图像识别可乐。
6. 输入 `voice add milk` 模拟语音加入牛奶。
7. 输入 `voice price bread` 模拟语音查询价格。
8. 输入 `cart` 展示购物车和总价。
9. 输入 `checkout` 生成支付链接。

## 9. 后续升级路线

- 第一阶段：保持命令行演示，完成赛题逻辑闭环。
- 第二阶段：接入摄像头帧采集和二维码识别。
- 第三阶段：训练 10 类商品分类模型，转换为 RKNN，替换关键词模拟识别。
- 第四阶段：增加 LVDS 屏幕 GUI，例如 Qt、LVGL 或 HTML 本地界面。
- 第五阶段：接入真实语音识别和支付后端。
