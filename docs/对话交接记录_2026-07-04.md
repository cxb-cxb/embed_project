## 遇到的问题与解决方法

原因：用户当时在 `C:\Users\33591>` 目录下运行相对路径，项目目录不在当前位置。


或直接使用完整路径：

```powershell

```

### 问题 2：板端脚本出现 `set: -\r: invalid option`

原因：Shell 脚本使用了 Windows CRLF 换行，板端 `sh` 不能正确解析。

解决方式：

- 已把脚本改为 Linux LF 换行。
- 新增 `.gitattributes`：

```text
*.sh text eol=lf
```

避免队友拉取后脚本再次变为 CRLF。

### 问题 3：板端没有 `qr_scanner` 可执行文件

原因：板端没有本地编译器，初始只推了源码，没有二进制。

解决方式：

使用 Docker Debian 容器交叉编译：

```powershell
docker run --rm -v "${PWD}:/work" -w /work debian:bookworm bash -lc "apt-get update >/dev/null && apt-get install -y gcc-aarch64-linux-gnu libc6-dev-arm64-cross make >/dev/null && mkdir -p build_arm && aarch64-linux-gnu-gcc -std=c11 -O2 -Wall -D_GNU_SOURCE -I lib/quirc src/qr_scanner.c lib/quirc/*.c -lm -o build_arm/qr_scanner"
```

推送：

```powershell
.\tools\adb\adb.exe push .\build_arm\qr_scanner /userdata/Embed_project/bin/qr_scanner
.\tools\adb\adb.exe shell "chmod +x /userdata/Embed_project/bin/qr_scanner"
```

### 问题 4：LVDS 屏幕坏了

解决方式：

不再依赖 LVDS，新增电脑端预览链路：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\view_camera_adb.ps1
```

当前电脑端方案默认使用逐帧 `adb pull`，帧率低于板端 21 fps，但稳定可用，适合调试和演示。

### 问题 5：GitHub 推送 403 权限错误

现象：

```text

```

原因：Windows Git Credential Manager 使用的是旧账号 `sequela753-hub`，而仓库 collaborator 添加的是 `sequela001`。

解决方式：

用 `sequela001` 重新登录 Git Credential Manager：

```powershell

```
