# 最终演示流程与应急卡

## 正式演示入口

建议打开两个 PowerShell 窗口。

窗口 1：启动视觉识别和 LVDS 叠加。

```powershell
cd "C:\Users\33591\Documents\嵌赛 2"
.\start_visual_lvds_demo.ps1
```

默认使用连续摄像头流，LVDS 会持续刷新。如果连续流异常，可临时回退：

```powershell
.\start_visual_lvds_demo.ps1 -SingleShot
```

窗口 2：启动语音查询。

```powershell
cd "C:\Users\33591\Documents\嵌赛 2"
.\start_voice_mic_query_loop.ps1 -Seconds 6 -ReplyMode auto
```

## 演示步骤

1. 将商品正面对准摄像头。
2. LVDS 屏幕显示商品识别结果和置信度。
3. 在语音窗口按回车。
4. 对板载麦克风说：

```text
这个多少钱
这个还有库存吗
推荐一下这个商品
```

5. 终端显示 ASR 识别文字和智能回复。
6. LVDS 信息区显示最近一次语音问题和回复。

## 推荐讲解话术

本项目实现了一个智能零售终端原型。系统通过摄像头识别当前商品，通过板载麦克风接收顾客问题，使用火山引擎豆包语音完成 ASR，再结合本地商品库和 DeepSeek 智能回复，实现“看见商品、听懂问题、自动回答”的闭环。网络或大模型不可用时，系统会自动回退到离线模板回复，保证现场演示稳定。

## 应急处理

### ADB 找不到设备

```powershell
adb devices -l
```

如果没有 `device`，重新插拔 USB，确认开发板已正常启动。

### LVDS 没画面

先确认 LVDS 状态：

```powershell
adb shell "cat /sys/class/drm/card0-LVDS-1/status"
```

如果不是 `connected`，检查屏线、电源和背光。

### 语音没有识别

靠近板载麦克风，录音时间可加长：

```powershell
.\start_voice_mic_query_loop.ps1 -Seconds 8 -ReplyMode auto
```

### DeepSeek 或网络异常

切换到离线保底回复：

```powershell
.\start_voice_mic_query_loop.ps1 -Seconds 6 -ReplyMode offline
```

### 当前商品联动不对

确认视觉窗口正在持续输出类似：

```text
0001: milk 牛奶 confidence=0.990
```

语音脚本会读取：

```text
D:\qsm_embed_dataset\lvds_overlay_runtime\current_product.json
```

如果该文件不是最新，重启视觉脚本。
