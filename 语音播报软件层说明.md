# 语音播报软件层说明

当前暂时没有外接喇叭模块，因此语音回复采用三档输出方式：

```text
terminal  只在终端显示回复，并保存最近一次回复文本
wav       终端显示 + 在电脑生成最近一次回复 WAV 文件
board     终端显示 + 生成 WAV + 推送到开发板播放
```

默认模式是 `terminal`，不会尝试播放声音。

## 常用命令

最终演示默认运行：

```powershell
.\start_final_demo.ps1
```

如果要同时生成电脑端 WAV 文件：

```powershell
.\start_final_demo.ps1 -VoiceOutput wav
```

单独测试语音问答：

```powershell
.\start_voice_mic_query_demo.ps1 -AsrText "牛奶多少钱" -VoiceOutput terminal
```

生成 WAV 测试：

```powershell
.\start_voice_mic_query_demo.ps1 -AsrText "牛奶多少钱" -VoiceOutput wav
```

输出文件位置：

```text
D:\qsm_embed_dataset\voice_reply_output\last_reply.txt
D:\qsm_embed_dataset\voice_reply_output\last_reply.json
D:\qsm_embed_dataset\voice_reply_output\last_reply_sapi.wav
```

## 后续接入喇叭

接入耳机、有源音箱或喇叭模块后，可以切到：

```powershell
.\start_voice_mic_query_demo.ps1 -AsrText "牛奶多少钱" -VoiceOutput board
```

该模式会先生成 WAV，再推送到开发板，并执行：

```bash
amixer -c 0 cset name='Playback Path' 'SPK_HP'
aplay -D hw:0,0 /tmp/qsm_last_voice_reply.wav
```

如果仍然无声，优先检查喇叭/耳机是否实际接入，以及输出路径 `SPK`、`HP`、`SPK_HP` 是否和硬件接口匹配。
