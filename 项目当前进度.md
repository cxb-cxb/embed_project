### 阶段 3：解决板端运行问题

当前问题：

- 板端暂未检测到 `g++`、`gcc`、`cmake`。
- 因此不能直接在板子上编译 C++ 项目。

可选方案：

1. 在板端安装编译工具链。
2. 使用 Linux 主机或 WSL 交叉编译 aarch64 程序。
3. 使用厂商 SDK 提供的交叉编译工具链。

推荐方案：

- 使用厂商 SDK 或 Ubuntu 交叉编译工具链生成 aarch64 可执行文件。
- 再用 adb push 到开发板运行。

示例：

```bash
aarch64-linux-gnu-g++ -std=c++17 -Iinclude src/*.cpp -o build/embed_project
adb push build/embed_project /userdata/Embed_project/build/embed_project
adb shell chmod +x /userdata/Embed_project/build/embed_project
adb shell /userdata/Embed_project/build/embed_project /userdata/Embed_project/data/products.csv
```

### 阶段 4：接入真实摄像头识别

目标：

- 从“模拟识别”升级为“真实摄像头输入”。

建议路线：

1. 先保持当前 GStreamer 摄像头预览命令可用。
2. 用 OpenCV 或 GStreamer 抓取摄像头帧。
3. 增加二维码或条码识别。
4. 把识别结果传给当前 `RetailRecognizer` 或直接查 `Catalog`。
5. 商品识别成功后调用 `cart.add(product)`。

建议新增文件：

```text
include/camera_capture.hpp
src/camera_capture.cpp
include/barcode_scanner.hpp
src/barcode_scanner.cpp
```

### 阶段 5：接入 RKNN 商品分类模型

目标：

- 满足赛题端侧 AI 要求。
- 实现 10 类商品外观识别。

建议流程：

1. 拍摄或收集 10 类商品图片。
2. 训练分类模型，例如 MobileNet、YOLO 分类头或轻量 CNN。
3. 转换为 RKNN 模型。
4. 在板端使用 RKNN runtime 推理。
5. 推理结果映射到 `Product.id`。
6. 返回 `RecognitionResult` 并加入购物车。

建议新增文件：

```text

src/rknn_product_detector.cpp
models/product_classifier.rknn
models/labels.txt
```

## 7. 当前风险与注意事项

### 7.1 板端缺少编译器

当前开发板上只检测到 `make`，没有检测到：

```text
gcc
g++
cmake
clang
```

因此后续要么安装板端编译器，要么使用交叉编译。

### 7.2 当前识别还是模拟逻辑

当前商品识别可以演示完整业务流程，但还不是真实 AI 识别。

比赛前必须至少完成其中一种真实识别：

- 条码识别
- 二维码识别
- RKNN 商品外观分类

如果时间紧，建议优先做二维码或条码识别，因为实现速度快、稳定性高。

如果要突出端侧 AI，建议接入 RKNN 商品分类模型。

### 7.3 支付链接是演示链接

当前支付 URL 不是正式支付接口。

如果比赛不要求真实支付，可以把它作为“支付二维码内容”展示。

如果要求真实支付，需要接入后端或真实支付平台。

### 7.4 WSL 有 PATH 警告

使用 WSL 编译时，会出现一些 Windows 路径翻译警告，例如：

```text
wsl: Failed to translate ...
```

目前不影响编译和测试。
